# Source Analysis Rubric

Use one pass per source. Do not merge sources during analysis.

## Required sections

### Source identity

- Title or URL.
- Type: website, guide, transcript, paper, docs, video, internal note.
- Why it matters for the module.

### Thesis

State the source's main claim in 2-5 sentences.

### Useful ideas

Extract ideas that should influence the course. Prefer principles, patterns, warnings, definitions, examples, and evaluation criteria.

### Risks and misunderstandings

Identify what a learner could misunderstand if the source were taught poorly.

### Pedagogical decisions

Convert source ideas into course design decisions. Example: "Because the source emphasizes activation, the module must include positive and negative eval prompts."

### Where it maps

List which module file should use the source:

- `01-fundamentos.md`
- `02-syllabus.md`
- `03-lecciones.md`
- checklist/evals

## Quality bar

A good analysis is opinionated. It does not merely summarize. It explains what the course should do differently because this source exists.
