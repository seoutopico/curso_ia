# Lesson Design Rubric

Use this when writing or reviewing `03-lecciones.md`.

## Lesson contract

Every lesson must include:

- What we are doing.
- Why it matters.
- Worked example.
- Template.
- Deliverable.
- Quality check.

## Flow

Lessons should accumulate. A learner should be able to see the artifact growing from lesson to lesson.

Recommended order:

1. Understand the problem.
2. Define scope.
3. Define use cases and non-use cases.
4. Write the activation or description layer.
5. Write core instructions.
6. Decide what belongs in references, assets, or scripts.
7. Add one reference if needed.
8. Add one asset if needed.
9. Add one script if deterministic validation helps.
10. Integrate the package.
11. Evaluate.
12. Prune and split.

## Failure signs

- A lesson explains but produces nothing.
- The example becomes a whole business system.
- Templates appear without filled examples.
- Evals test only happy paths.
- Resources are mentioned but not integrated into the workflow.
